Full security audit performed 2026-03-22 for App Store submission readiness.

## Key Findings
- P0: Anthropic API key and Supabase service role key in local .env (NOT in git — verified)
- P1: Health data not encrypted at column level (Supabase at-rest encryption covers basic case)
- P1: No certificate pinning in mobile app
- P1: No jailbreak/root detection
- P2: Docker compose had default DB credentials (FIXED — now requires explicit env vars)
- P2: CORS defaulted to wildcard (FIXED — now defaults to localhost:8081)

## Positive Controls Verified
- JWT type distinction (access/refresh), token blacklisting, refresh rotation
- Brute-force protection (5 attempts, 15min lockout)
- Rate limiting on auth endpoints
- PBKDF2-SHA256 with 600k rounds
- SecureStore for tokens on mobile
- GDPR export and soft-delete endpoints exist
- Docker prod: read-only fs, no-new-privileges, cap_drop ALL
- Security headers middleware (XSS, HSTS, CSP)
- No SQL injection (all parameterized queries)
- No IDOR (all queries scoped to current_user.id)

**Why:** App Store submission requires demonstrable security for health data apps.
**How to apply:** Reference this audit for pre-launch security checklist. P0s require key rotation before production deployment.
